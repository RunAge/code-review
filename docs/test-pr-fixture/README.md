# Test PR Fixture

This folder exists only for testing review workflows in the app.

## Why this fixture exists

- It contains small, predictable code changes.
- It creates useful inline-comment targets.
- It is safe to keep outside runtime code.

## Suggested review checks

1. Add inline comment on a single line.
2. Add range comment from one line to another.
3. Add a suggestion block in comment.
4. Submit `COMMENT` review with pending comments.
